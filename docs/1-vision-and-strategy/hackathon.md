# Gemini Live Agent Challenge

**Redefining Interaction: From Static Chatbots to Immersive Experiences**

The challenge invites builders to "stop typing, and start interacting" by creating next-generation AI agents that move beyond text-based interfaces. Participants are encouraged to leverage multimodal AI capabilities to build immersive, real-time experiences.

## Key Details

- **Deadline:** March 16, 2026 @ 5:00pm PDT
- **Prize Pool:** $80,000 in total cash prizes
- **Participants:** 3,013 registered
- **Format:** Online, public competition
- **Sponsor:** Google (Gemini)
- **Managed by:** Devpost

## Eligibility

- Must be above the legal age of majority in country of residence
- Specific countries/territories excluded per full rules

## Three Challenge Categories

### 1. Live Agents

Focus on real-time interaction using audio and vision capabilities. Build agents users can naturally converse with and that handle interruptions gracefully. Examples include real-time translators, vision-enabled tutors, or customer support voice agents.

**Required:** Gemini Live API or ADK (Agent Development Kit), hosted on Google Cloud.

### 2. Creative Storyteller

Create agents that seamlessly blend text, images, audio, and video outputs in a single stream. Applications include interactive storybooks, marketing asset generators, educational explainers, and social content creators.

**Required:** Gemini's interleaved/mixed output capabilities, hosted on Google Cloud.

### 3. UI Navigator

Build agents that observe browser/device displays, interpret visual elements, and execute user-directed actions. Applications include universal web navigators, workflow automators, or visual QA testing agents.

**Required:** Gemini multimodal for screenshot interpretation, hosted on Google Cloud.

## Universal Requirements

All projects must:

1. Leverage a Gemini model
2. Use Google GenAI SDK or ADK
3. Deploy backend on Google Cloud

## Submission Requirements

1. **Text Description:** Project features, technologies, data sources, and learnings
2. **Public Code Repository:** Include spin-up instructions for reproducibility
3. **Google Cloud Deployment Proof:** Screen recording or code demonstrating GCP service usage
4. **Architecture Diagram:** Visual system representation
5. **Demonstration Video:** Under 4 minutes showing multimodal features in real-time with problem/solution pitch

## Bonus Opportunities
